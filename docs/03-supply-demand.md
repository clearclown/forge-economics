## 3.1 従来の経済学では

### 需要曲線——「買いたい量」と「価格」の関係

**需要曲線**は、「ある商品の価格が変わったとき、
消費者がどれだけ買いたいか」を表すグラフです。

基本原則は単純です：

```
  価格が上がる → 買いたい人が減る（需要減少）
  価格が下がる → 買いたい人が増える（需要増加）
```

これを「需要の法則」と呼びます。グラフにするとこうなります：

```
  価格
  (円)
   ^
   |
  500 ─ *
   |      *
  400 ─     *
   |          *
  300 ─         *
   |              *
  200 ─             *
   |                  *
  100 ─                 *  *  *
   |
   +──────────────────────────────→ 数量(個)
       1   2   3   4   5   6   7

  ※ 右下がりの曲線 = 需要曲線
  ※ 価格が安いほど、買いたい量が多い
```

**なぜ右下がりか？** 第 1 章で学んだ「限界効用逓減の法則」が理由です。

```
  パンを 1 個食べる → 満足度大 → ¥500 でも買う
  パンを 2 個食べる → そこそこ  → ¥300 なら買う
  パンを 3 個食べる → もう十分  → ¥100 でも要らない

  → 追加の 1 個に対する支払意思額が下がっていく
  → だから需要曲線は右下がり
```

### 供給曲線——「売りたい量」と「価格」の関係

**供給曲線**は、「ある商品の価格が変わったとき、
生産者がどれだけ売りたいか」を表すグラフです。

基本原則はこうです：

```
  価格が上がる → 売りたい人が増える（供給増加）
  価格が下がる → 売りたい人が減る（供給減少）
```

これを「供給の法則」と呼びます：

```
  価格
  (円)
   ^
   |
  500 ─                       *  *
   |                     *
  400 ─                *
   |              *
  300 ─         *
   |        *
  200 ─   *
   |    *
  100 ─ *
   |
   +──────────────────────────────→ 数量(個)
       1   2   3   4   5   6   7

  ※ 右上がりの曲線 = 供給曲線
  ※ 価格が高いほど、売りたい量が多い
```

**なぜ右上がりか？** 追加で 1 個作るコスト（限界費用）が
増えていくからです。

```
  パンを 1 個焼く → 材料費だけ     → ¥100 で採算が合う
  パンを 5 個焼く → 残業が必要     → ¥300 でないと割に合わない
  パンを 10 個焼く → 設備増設が必要 → ¥500 でないと赤字

  → 追加の 1 個を作るコストが上がっていく
  → だから供給曲線は右上がり
```

### 均衡価格——二つの曲線が交わるところ

需要曲線と供給曲線を同じグラフに描くと、
一つの交点があります。これが**均衡価格**です：

```
  価格
  (円)
   ^
   |
  500 ─ *                         * *  ← 供給曲線
   |      *                   *
  400 ─     *             *
   |          *       *
  300 ─  - - - - * - - - - - - - - - -  ← 均衡価格 ¥300
   |          *   *                       均衡数量 4 個
  200 ─     *       *
   |    *             *
  100 ─ *               *  *  *  ← 需要曲線
   |
   +──────────────────────────────→ 数量(個)
       1   2   3   4   5   6   7
                   ^
                   均衡数量
```

**均衡価格の意味：**

```
  価格が ¥300 のとき：
    買いたい量（4 個）= 売りたい量（4 個）
    → 過不足なし → 市場は安定

  価格が ¥500 のとき：
    買いたい量（1 個）< 売りたい量（7 個）
    → 売れ残りが出る → 売り手が価格を下げる → ¥300 に近づく

  価格が ¥100 のとき：
    買いたい量（7 個）> 売りたい量（1 個）
    → 品薄になる → 買い手が競い合う → ¥300 に近づく

  → 市場は自然に均衡価格に向かう
  → これがアダム・スミスの「見えざる手」
```

### 現実の市場の問題

教科書ではきれいに均衡が成立しますが、
現実の市場には多くの問題があります：

```
  ┌─────────────────────────────────────────────────────┐
  │  現実の市場が理想から乖離する理由                     │
  │                                                     │
  │  [1] 情報の非対称性                                  │
  │      → 売り手は品質を知っているが、買い手は知らない   │
  │      → 例：中古車（レモン市場問題）                   │
  │                                                     │
  │  [2] 外部性                                          │
  │      → 取引当事者以外に影響が及ぶ                    │
  │      → 例：工場の公害（負の外部性）                   │
  │                                                     │
  │  [3] 独占・寡占                                      │
  │      → 少数の企業が価格を操作できる                  │
  │      → 例：OPEC（石油カルテル）                       │
  │                                                     │
  │  [4] 感情的行動                                      │
  │      → 人間はパニックや熱狂で非合理に行動する         │
  │      → 例：バブル経済、パニック売り                   │
  └─────────────────────────────────────────────────────┘
```

## 3.2 完全競争市場の 5 条件

経済学の教科書には、「もし市場がこの条件を満たせば、
資源は最も効率的に配分される」という理想的な市場の定義があります。
それが**完全競争市場**です。

5 つの条件があります：

```
  ┌─────────────────────────────────────────────────────┐
  │                完全競争市場の 5 条件                  │
  │                                                     │
  │  [1] 多数の売り手と買い手                            │
  │      → 一人の行動が市場全体の価格に影響しない         │
  │                                                     │
  │  [2] 同質の財（homogeneous goods）                   │
  │      → 誰が作っても同じ品質                          │
  │                                                     │
  │  [3] 完全情報                                        │
  │      → 全員が価格・品質の情報を知っている             │
  │                                                     │
  │  [4] 自由な参入・退出                                │
  │      → いつでも市場に入れて、いつでも出られる         │
  │                                                     │
  │  [5] 合理的行動                                      │
  │      → 全員が自分の利益を最大化するように行動する     │
  └─────────────────────────────────────────────────────┘
```

経済学者はこの条件が現実にはほぼ成り立たないことを承知で、
「理想的な基準点（ベンチマーク）」として使っています。

では、Forge の CU 市場はどうでしょうか？

## 3.4 CU 価格決定メカニズム

### グローバルなオーダーブックがない世界

従来の市場（証券取引所など）には、
すべての売買注文が集まる「オーダーブック（注文板）」があります：

```
  従来の市場：
  ─────────────────────────
  売り注文: ¥301 × 100 株、¥302 × 200 株、...
  買い注文: ¥300 × 150 株、¥299 × 300 株、...
  → 中央のオーダーブックが全注文を管理
  → 売値と買値が一致したら約定
```

Forge にはこのような中央管理の注文板がありません。
では、価格はどうやって決まるのでしょうか？

### 各ノードがローカルに価格を決定する

```
  ┌──────────────────────────────────────────────────────┐
  │  CU 価格決定の仕組み                                 │
  │                                                      │
  │  各ノードが独立に以下を観察する：                      │
  │                                                      │
  │  [1] 自分が受け取るリクエスト数                        │
  │      → 需要の代理変数                                 │
  │      → リクエストが多い = 需要が高い                   │
  │                                                      │
  │  [2] ネットワーク内のアクティブノード数                │
  │      → 供給の代理変数                                 │
  │      → ノードが多い = 供給が多い                       │
  │                                                      │
  │  この二つの情報から、各ノードが自分の価格を計算：       │
  │                                                      │
  │  effective_price =                                    │
  │      base_cu_per_token × (demand_factor / supply_factor) │
  │                                                      │
  └──────────────────────────────────────────────────────┘
```

### EMA（指数移動平均）による平滑化

生の需要・供給データは秒単位で変動します。
これをそのまま価格に反映すると、価格が激しく振動してしまいます。

そこで**EMA（Exponential Moving Average、指数移動平均）**を使って
平滑化します：

```
  EMA とは？
  ─────────────────────────
  「最新のデータを重視しつつ、過去のデータも考慮する平均」

  普通の平均：  (a + b + c + d + e) / 5
  EMA：        最新のデータほど大きな重みをつける

  例（直近 5 期間の需要データ）：
    100, 110, 105, 120, 130

  単純平均： (100+110+105+120+130)/5 = 113
  EMA：     最新の 130 に大きな重み → 約 120

  → 急激な変動を抑えつつ、トレンドには追随する
  → 1 件の異常なリクエストで価格が急変しない
```

```
  価格
   ^
   |         *                        ← 生データ（激しく振動）
   |    *  *   *  *
   |  *          *  *     *  *
   | *                 * *     * *
   |                              *
   |   ____----____----____----   ← EMA（滑らかな曲線）
   |
   +────────────────────────────────→ 時間

  EMA を使うことで、価格は安定しつつも
  需給の変化には追随する
```

### ゴシッププロトコルによる価格収束

各ノードが独立に計算した価格は、最初はバラバラです。
これを一つの市場価格に収束させるのが
**ゴシッププロトコル**です。

```
  ┌──────────────────────────────────────────────────────┐
  │  ゴシッププロトコルの仕組み                            │
  │                                                      │
  │  [Step 1] 各ノードが自分の価格を計算                  │
  │                                                      │
  │    ノード A: 3.2 CU/token                             │
  │    ノード B: 2.8 CU/token                             │
  │    ノード C: 3.0 CU/token                             │
  │    ノード D: 3.1 CU/token                             │
  │                                                      │
  │  [Step 2] ランダムに選んだ隣接ノードに自分の価格を伝える │
  │                                                      │
  │    A → B: 「うちは 3.2 だよ」                          │
  │    C → D: 「うちは 3.0 だよ」                          │
  │                                                      │
  │  [Step 3] 受け取った情報を自分の計算に反映             │
  │                                                      │
  │    B: 「A は 3.2 か。自分は 2.8 だけど、少し上げよう」  │
  │    → B は 2.9 に修正                                   │
  │                                                      │
  │  [Step 4] これを繰り返す                               │
  │                                                      │
  │    A: 3.2 → 3.1 → 3.05 → 3.02 → ...                  │
  │    B: 2.8 → 2.9 → 2.95 → 2.98 → ...                  │
  │    C: 3.0 → 3.0 → 3.01 → 3.01 → ...                  │
  │    D: 3.1 → 3.05 → 3.02 → 3.01 → ...                 │
  │                                                      │
  │  → 全ノードの価格が約 3.0 に収束！                     │
  └──────────────────────────────────────────────────────┘
```

この仕組みは、フリードリッヒ・ハイエク（1899-1992）が
提唱した**「分散的知識」**の理想的な実装です。

ハイエクは 1945 年の論文で、こう主張しました：

```
  「社会全体の知識は、一つの頭脳に集中できない。
   知識は無数の個人に分散しており、
   価格メカニズムがこの分散した知識を集約する。」

  → Forge はまさにこれをプロトコルレベルで実現
  → 各ノードが持つローカルな情報（需要・供給の観測）が
    ゴシップを通じて価格に集約される
  → 中央の「全知の計画者」は不要
```

## 3.5 モデルティア別の価格表

CU の基本価格はすべての推論で同じではありません。
モデルのサイズ（パラメータ数）に応じて、ティア（段階）が設定されています：

```
  ┌──────────┬──────────────┬──────────────┬─────────────────┐
  │ ティア    │ パラメータ数  │ 基本 CU/token │ モデル例         │
  ├──────────┼──────────────┼──────────────┼─────────────────┤
  │ Small    │ < 3B         │      1       │ Qwen 2.5 0.5B   │
  │ Medium   │ 3B - 14B     │      3       │ Qwen 3 8B       │
  │ Large    │ 14B - 70B    │      8       │ DeepSeek V3     │
  │ Frontier │ > 70B        │     20       │ Llama 3.1 405B  │
  └──────────┴──────────────┴──────────────┴─────────────────┘
```

**なぜティアが必要か？**

```
  大きなモデル：
  ─────────────────────────
  ・多くのパラメータ → 多くの計算が必要 → 多くの電力
  ・高品質の推論を生成できる
  ・当然、コスト（CU）も高くなるべき

  小さなモデル：
  ─────────────────────────
  ・少ないパラメータ → 少ない計算 → 少ない電力
  ・品質は大きなモデルに劣る場合がある
  ・コスト（CU）は安くあるべき

  → モデルの計算量に比例した価格設定
  → これがティア制
```

### MoE モデルの特殊ケース

**MoE（Mixture of Experts）** は、パラメータ数は大きいが
実際に使われるのは一部だけ、というアーキテクチャです：

```
  例: Qwen 3 30B-A3B
  ─────────────────────────
  総パラメータ数：30B（300 億）
  活性パラメータ数：3B（30 億）← 推論時に実際に使う量

  課金基準は？
  → 活性パラメータ数（3B）で判定
  → Medium ティア（3 CU/token）

  なぜ？
  → 実際に使う計算量が 3B 相当だから
  → 「社会的に必要な計算量」に基づく価格設定
  → 第 1 章の労働価値説の考え方と一致
```

### 最終価格の計算例

```
  エージェント B が Qwen 3 8B（Medium ティア）に
  1,000 トークンの推論をリクエストする場合：

  ──────────────────────────────────────
  基本価格：3 CU/token × 1,000 tokens = 3,000 CU

  現在の需給状況：
    demand_factor = 1.2（需要がやや高め）
    supply_factor = 1.0（供給は平常）

  最終価格：
    3,000 × (1.2 / 1.0) = 3,600 CU
  ──────────────────────────────────────

  需要が高いので、基本価格より 20% 高くなっている。
  これは「繁忙期の価格上昇」に相当する。
```
